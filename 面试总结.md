**你这个code agent具体是做什么工作的**

我们做的这个agent其实是code agent下的一个子任务，issue localization。核心任务是给定一个 Issue Report 的自然语言描述，自动在代码库中找到需要修改的具体位置——包括文件、类、函数三个**粒度**。

**为什么要用图，传统的比如关键字检索的方法可行嘛(你们使用了哪些工具)**

传统的方法有哪些：BM25关键字搜索；用RAG的思想进行一个代码向量数据库的构造；使用 Agent 或者 Agentless 方式

传统的方法有哪些不足：

我们在这里把握的一个要点就是：issue 描述的是自然语言，代码库是结构化程序，两者之间存在天然的语义鸿沟。从报错位置到根源，需要跨越多个文件、沿着调用链追溯——这就是所谓多跳推理(multi-hopreasoning)问题。

**图工具有哪些，各自有什么作用**

将代码库解析为有向异构图，图中包含 directory、file、class、function 四种节点以及 contains、imports、invokes、inherits 四种有向边，将代码的结构依赖关系显式编码为可查询的图结构。
